# Sub-Component 7.7: Reader & Editor

## Purpose
Read generated chapters and edit them with AI assistance.

---

## Requirements Gathering

### Question 1: Reading Mode

**Reading mode — distraction-free or with controls?**

Options:
- **A. Distraction-free:** Clean, minimal UI
- **B. With controls:** Navigation always visible
- **C. Toggle:** User switches modes
- **D. Auto-hide:** Controls fade when not active
- **E. Contextual:** Controls based on activity

### Question 2: Editing Mode

**Editing — inline or separate edit mode?**

Options:
- **A. Inline:** Edit directly in reader
- **B. Separate mode:** Switch to edit view
- **C. Selection-based:** Click to edit section
- **D. Side-by-side:** Reader | Editor split
- **E. Modal:** Popup editor

### Question 3: AI Assistance

**AI assistance — always visible or on-demand?**

Options:
- **A. Always visible:** Persistent panel
- **B. On-demand:** Click to open
- **C. Contextual:** Right-click options
- **D. Floating:** Follows cursor
- **E. Inline suggestions:** Underlined suggestions

### Question 4: Version Control

**Version control — prominent or hidden?**

Options:
- **A. Prominent:** Always visible timeline
- **B. Accessible:** Easy to find
- **C. Menu-based:** In menu/options
- **D. Auto-save:** Background versioning
- **E. Hidden:** Power user feature

### Question 5: Navigation

**Chapter navigation style?**

Options:
- **A. Continuous scroll:** One long page
- **B. Pagination:** Click next/previous
- **C. Chapter list:** Sidebar with chapters
- **D. Progress bar:** Scrub through story
- **E. Hybrid:** Default with options

### Question 6: Typography

**Typography control level?**

Options:
- **A. Fixed:** No user control
- **B. Size only:** Adjust font size
- **C. Full control:** Size, font, spacing, theme
- **D. Presets:** Choose from presets
- **E. Smart:** Adapts to device/content

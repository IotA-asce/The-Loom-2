# Sub-Component 6.2: Outline Architect

## Purpose
Design the structure of each chapter before writing — scene breakdown, pacing, emotional beats, and narrative arc.

---

## Requirements Gathering

### Question 1: Outline Visibility

**Should outlines be shown to user before writing?**

Options:
- **A. Always:** User must approve outline first
- **B. Optional:** User can choose to see or skip
- **C. On request:** User asks to see outline
- **D. Never:** Generate chapter directly
- **E. Smart:** Show for complex chapters, skip for simple

Considerations:
- Seeing outline gives user control
- But adds step to generation flow
- Some users may prefer surprise

### Question 2: Outline Detail Level

**How detailed should outlines be?**

Options:
- **A. Minimal:** Scene list only (title + characters)
- **B. Standard:** Scene purpose + key events
- **C. Detailed:** Full paragraph per scene
- **D. Very detailed:** Dialogue snippets, specific beats
- **E. Adjustable:** User chooses detail level

Considerations:
- More detail = better guidance for writing
- But may constrain creativity
- Balance structure and flexibility

### Question 3: User Outline Modification

**Should user be able to modify outlines?**

Options:
- **A. Full editing:** User can change anything
- **B. Reordering:** User can reorder scenes
- **C. Add/remove:** User can add or cut scenes
- **D. Suggestions:** User can request changes
- **E. No modification:** Outline is AI's plan only

Considerations:
- User control vs. AI expertise
- Some users may have specific ideas
- But AI may have better structural sense

### Question 4: Chapter Structure Variety

**How do we ensure variety in chapter structures?**

Options:
- **A. Template-based:** Rotate through templates
- **B. Genre-appropriate:** Structure fits genre conventions
- **C. Plot-driven:** Structure serves story needs
- **D. Random variation:** Randomize to avoid patterns
- **E. Original-matching:** Mirror original manga's structure variety

Considerations:
- Variety keeps reading interesting
- But some structures serve story best
- Original manga's rhythm may be good guide

### Question 5: Emotional Arc Planning

**How explicitly should emotional beats be planned?**

Options:
- **A. No planning:** Let emotion emerge naturally
- **B. Major beats only:** Plan big emotional moments
- **C. Scene-by-scene:** Every scene has target emotion
- **D. Character-based:** Plan per-character emotional journey
- **E. Full arc:** Plan complete emotional roller coaster

Considerations:
- Emotional pacing critical for engagement
- Too planned may feel manipulative
- But aimless chapters feel flat

### Question 6: Cliffhanger Strategy

**How should chapter endings be planned?**

Options:
- **A. Always cliffhanger:** Every chapter ends with hook
- **B. Varied endings:** Mix of resolution and hooks
- **C. Arc-appropriate:** Cliffhangers at act breaks
- **D. User preference:** User selects ending type
- **E. Story-driven:** Whatever serves story best

Considerations:
- Cliffhangers drive continued reading
- But constant tension is exhausting
- Some chapters need resolution

interface ChapterOutline {
  chapterNumber: number;
  title: string;
  summary: string;
  
  // Structure
  scenes: SceneOutline[];

  estimatedReadTime: number;

}

interface SceneOutline {
  sequence: number;
  title: string;
  purpose: 'setup' | 'rising-action' | 'climax' | 'falling-action' | 'resolution';
  
  // Content
  summary: string;
  keyEvents: string[];
  
  // Characters
  characters: string[];
  location: string;
  
  // Emotional target
  emotionalBeat: string;
  tension: 'low' | 'medium' | 'high';
}
